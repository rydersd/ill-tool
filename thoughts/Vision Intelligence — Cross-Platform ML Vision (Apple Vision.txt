Vision Intelligence — Cross-Platform ML Vision (Apple Vision + ONNX/CUDA)
Context
IllTool's tracing pipeline relies on external tools: vtracer via popen() for SVG contours, Python backends for normal maps, and a hand-rolled C++ VisionEngine for Canny/Hough/contour operations. There is no Apple Vision framework or ML inference integration — the existing VisionEngine.h/.cpp is pure C++ with no platform-specific ML.

The goal: add GPU-accelerated instance segmentation (click-to-select subjects), native contour detection, and body/face/hand pose extraction. Must work on macOS (Apple Silicon + Intel) AND Windows (NVIDIA CUDA). On macOS, prefer Apple Vision (zero config, no model downloads). On Windows, use ONNX Runtime with CUDA/DirectML.

Runtime selection (in VisionIntelligence.cpp):
  macOS + Apple Silicon → AppleVisionBridge (preferred)
  macOS + Intel         → AppleVisionBridge for CPU ops,
                          OnnxVisionBridge for Neural Engine ops
  Windows               → OnnxVisionBridge (CUDA or DirectML)

The consumer code (TraceModule, panels) only calls VisionIntelligence.h — never touches backends directly.

Common Interface — VisionIntelligence.h
#ifdef __cplusplus
extern "C" {
#endif

// --- Capabilities ---
typedef enum {
    VI_BACKEND_NONE        = 0,
    VI_BACKEND_APPLE_VISION = 1,  // macOS Vision.framework
    VI_BACKEND_ONNX_CUDA   = 2,  // ONNX Runtime + CUDA
    VI_BACKEND_ONNX_COREML = 3,  // ONNX Runtime + CoreML
    VI_BACKEND_ONNX_DIRECTML = 4, // ONNX Runtime + DirectML
    VI_BACKEND_ONNX_CPU    = 5,  // ONNX Runtime CPU fallback
} VIBackend;

VIBackend VIGetActiveBackend();
bool VIHasNeuralEngine();        // Apple Silicon or CUDA GPU available
bool VIHasInstanceSegmentation(); // SAM or VNInstanceMask available
bool VIHasPoseDetection();       // RTMO or VNBodyPose available
bool VIHasDepthEstimation();     // Depth Anything V2 ONNX loaded

// --- Contour Detection ---
typedef struct {
    double* points;     // interleaved x,y (normalized 0-1)
    int pointCount;
    bool closed;
} VIContour;

int VIDetectContours(
    const char* imagePath,
    float contrastAdjustment,  // 0.0-3.0
    float contrastPivot,       // 0.0-1.0
    bool detectDarkOnLight,
    VIContour** outContours
);
void VIFreeContours(VIContour* contours, int count);

// --- Instance Segmentation (SAM 2 / VNInstanceMask) ---
typedef struct {
    unsigned char* mask;   // grayscale (0/255), width*height
    int width, height;
    int instanceIndex;
    float score;           // confidence 0-1
} VIInstanceMask;

int VIDetectInstances(
    const char* imagePath,
    VIInstanceMask** outMasks
);
void VIFreeInstanceMasks(VIInstanceMask* masks, int count);

// Click-to-select: get mask for instance at (x,y) normalized coords
int VISelectInstanceAtPoint(
    const char* imagePath,
    float x, float y,           // click point (normalized 0-1)
    VIInstanceMask** outMask    // single mask for the selected instance
);

// --- Monocular Depth Estimation ---
// Returns normalized depth map (0.0=near, 1.0=far), width*height floats
bool VIEstimateDepth(
    const char* imagePath,
    float** outDepthMap,        // caller frees with VIFreeDepthMap
    int* outWidth, int* outHeight
);
void VIFreeDepthMap(float* depthMap);

// --- Body Pose ---
typedef struct {
    const char* jointName;   // e.g. "left_shoulder"
    float x, y;              // normalized 0-1
    float confidence;
} VIJoint;

int VIDetectBodyPose(const char* imagePath, VIJoint** outJoints);
void VIFreeJoints(VIJoint* joints, int count);

// --- Face Landmarks ---
typedef struct { float x, y; } VIFacePoint;

int VIDetectFaceLandmarks(
    const char* imagePath,
    VIFacePoint** outPoints  // 73 (Apple) or 468 (MediaPipe)
);
void VIFreeFacePoints(VIFacePoint* points, int count);

// --- Hand Pose ---
int VIDetectHandPose(const char* imagePath, VIJoint** outJoints);

// --- Lifecycle ---
bool VIInitialize();   // load models, init runtime
void VIShutdown();     // release resources

#ifdef __cplusplus
}
#endif

Phase 2.5: Monocular Depth -> Automatic Layer Decomposition
Both platforms via ONNX (Depth Anything V2). No Apple Vision equivalent — always uses ONNX Runtime regardless of platform. ~25MB model, <1s on GPU.

Why this is the radical feature
Layer separation is the most tedious manual step in reference-based illustration. No tool automates it. Depth Anything V2 produces a per-pixel depth map from a single photograph, which we quantize into discrete layers and trace separately.

How it compounds with existing modules
PerspectiveModule: Depth gradients corroborate VP placement. Steep depth falloff perpendicular to a detected VP confirms it's real. EstimateVanishingPoints() (VisionEngine.h:251) already produces VP candidates — depth ranks them by geometric consistency.
DecomposeModule: Depth layers are another decomposition axis. Paths in the same depth band auto-group via the same DrawOverlay colored-overlay pattern (DecomposeModule.cpp:283).
SurfaceModule: Depth discontinuities mark object boundaries. A sudden depth jump means "different surface" — eliminates ambiguity in InferSurfaceType() at object edges.
Instance segmentation (Phase 3): Separates what (subject identity), depth separates where (spatial position). Combined: "the character in the foreground" vs "the character in the background" on different layers automatically.

TraceModule additions
New method: void ExecuteDepthDecompose()
Flow:
FindImagePath() -> get placed image
VIEstimateDepth() -> normalized depth buffer (0.0=near, 1.0=far)
Histogram-cluster depth values into N bands (user slider, default 3-5)
Use Otsu's method or simple k-means on 1D depth histogram
For each depth band:
a. Create binary mask (pixels within depth range)
b. Apply mask to original image (zero outside)
c. Write masked image to temp
d. Run VIDetectContours() on masked image
e. Create named AI layer ("FG", "MG-1", "MG-2", "BG")
f. Place contours on the corresponding layer
Optionally: feed depth-derived layer boundaries to PerspectiveModule for VP ranking
DrawOverlay extension
When depth preview is active, draw depth map as color-coded overlay (red=near -> blue=far)
Each depth band drawn in its assigned color
Same annotator pattern as DecomposeModule cluster overlays

Phase 4: Masked Contour Tracing (the compound feature)
Click a subject -> isolate -> vectorize just that subject. No other tool does this in one workflow.

Implementation (~20 lines of glue)
In ExecuteAppleContours(), if instance mask composite is active:

Load original image via stb_image
Apply composite mask (zero pixels outside mask)
Write masked image to temp file
Run VIDetectContours() on masked image
Result: clean vector contours of just the selected subject(s)
Works identically on both backends — mask comes from SAM (Windows) or VNInstanceMask (macOS), contours come from VisionEngine::FindContours (Windows) or VNDetectContours (macOS).

Body Pose: Place person photo -> "Detect Pose" -> skeleton overlay -> "Commit" -> construction lines on layer
